# Telegram Bot Backend

> ⚠️ **Статус: в разработке.** Функциональность и API могут меняться.

Backend-сервис для Telegram-бота на Flask.

## Что делает

- При первом обращении пользователя — регистрирует его в базе и создаёт начальный склад (набор юнитов).
- Если пользователь уже есть в базе — возвращает актуальное состояние его склада.
- В фоне работает периодическая задача, которая обновляет данные о складе игрока независимо от запросов пользователей.
- Есть тестовый эндпоинт `Endpoints/cal_kestis_jedi.py` для примера. В дальнейшем можно будет выбирать конкретного
персонажа для сравнения

## Стек

- **Flask** — веб-фреймворк, обработка HTTP-запросов от бота
- **peewee** — ORM для работы с базой данных
- **APScheduler** — планировщик фоновых задач (обновление склада игроков)
- **loguru** — логирование
- **requests** — HTTP-запросы к внешним сервисам
- **Docker**  — контейнеризация и деплой приложения

## Фоновые задачи

Обновление персонажей (юнитов) происходит по расписанию через **APScheduler**, независимо от входящих запросов. Логика — в `Utils/update_units.py`.

## Логирование

Все логи пишутся через **loguru**.
